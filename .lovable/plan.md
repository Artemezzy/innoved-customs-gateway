
# План: Кликабельные блоки услуг с детальными страницами

## Обзор задачи
Создание системы детальных страниц для услуг. При клике на карточку услуги пользователь переходит на отдельную страницу с подробным описанием. Услуги используют только локальные статические данные (без Hygraph).

---

## Контент страницы "Оформление импорта" (из документа)

**SEO мета-теги:**
- Title: Оформление импорта с ИННОВЭД — круглосуточное электронное декларирование по России
- Description: Комплексное оформление импорта для бизнеса по всей России. Электронное декларирование, автоматизация, выделенный менеджер и скорость от ИННОВЭД.
- Keywords: оформление импорта, таможенное оформление, ВЭД-услуги, электронное декларирование, импорт товаров, сертификация, ИННОВЭД

**Структура контента:**
1. Вводный текст о важности правильного оформления импорта
2. Блок "Почему выбирают ИННОВЭД" (5 преимуществ)
3. Блок "Как проходит оформление импорта" (5 шагов)
4. Блок "Что вы получаете" (5 результатов)
5. Призыв к действию с кнопкой связи

## Технические детали

### 1. Создание файла данных услуг
**Новый файл:** `src/data/services.ts`

Статические данные для всех услуг с полями:
- `slug` — URL-идентификатор (import, export, hs-code и т.д.)
- `title` — название на ru/en
- `icon` — имя иконки Lucide
- `seo` — title, description, keywords для каждого языка
- `content` — структурированный контент страницы

Для "Оформление импорта" — полный контент из документа.
Для остальных 7 услуг — заглушки с минимальной информацией.

### 2. Рефакторинг хука useServices
**Файл:** `src/hooks/useServices.ts`

Упрощение: убрать зависимость от Hygraph, использовать только данные из `src/data/services.ts`. Добавить поле `slug` к каждой услуге.

### 3. Обновление типов
**Файл:** `src/services/hygraph.ts`

Добавление поля `slug` в интерфейс `HygraphService` для совместимости.

### 4. Кликабельные карточки
**Файл:** `src/components/ServiceCard.tsx`

Обернуть карточку в `Link` для навигации на `/services/{slug}`.

### 5. Обновление компонента Services
**Файл:** `src/components/Services.tsx`

Передача `slug` в компонент `ServiceCard`.

### 6. Создание страницы детальной услуги
**Новый файл:** `src/pages/ServiceDetailPage.tsx`

Структура (по аналогии с BlogPostPage):
- Hero-секция с названием услуги
- Кнопка "Назад к услугам"
- Динамические SEO мета-теги
- Основной контент с prose-стилями
- Призыв к действию

Для услуг без контента — заглушка "Подробное описание скоро будет добавлено" + кнопка связи.

### 7. Хук для получения детальной услуги
**Новый файл:** `src/hooks/useServiceDetail.ts`

Простой хук для получения данных услуги по slug из статических данных.

### 8. Добавление маршрута
**Файл:** `src/App.tsx`

Новый маршрут: `/services/:slug` → `ServiceDetailPage`

## Структура новых файлов

```text
src/
├── data/
│   └── services.ts           # Статические данные услуг
├── hooks/

    └── ServiceDetailPage.tsx # Страница детальной услуги
```

## Порядок реализации

1. Создать `src/data/services.ts` — полный контент для импорта, заглушки для остальных
2. Создать `src/hooks/useServiceDetail.ts`
3. Создать `src/pages/ServiceDetailPage.tsx`
4. Обновить `src/services/hygraph.ts` — добавить slug в интерфейс
5. Обновить `src/hooks/useServices.ts` — использовать локальные данные
6. Обновить `src/components/ServiceCard.tsx` — добавить кликабельность
7. Обновить `src/components/Services.tsx` — передать slug
8. Обновить `src/App.tsx` — добавить маршрут
