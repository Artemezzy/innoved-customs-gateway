

## Внедрение текстов пререндеринга в prerender-meta.mjs

### Что делаем

Добавляем поле `text` к каждой записи в массиве `pages` и выводим его как `<p>` внутри `<div id="root">` после заголовков и перед `<nav>`. Всего 54 страницы.

### Технические изменения

**Файл:** `scripts/prerender-meta.mjs`

1. **Добавить поле `text` к статическим страницам** (строки 110–248) — 11 страниц: `/`, `/about`, `/services`, `/how-we-work`, `/contact`, `/blog`, `/faq`, `/news`, `/privacy`, `/terms`, `/tamozhennyj-broker` и 4 кейса.

2. **Добавить поле `text` в `generateServicePages()`** — расширить массив `services` (строки 257–269) дополнительным полем `preText` для каждой из 11 услуг, использовать его при формировании объекта страницы.

3. **Добавить поле `text` в `generateCityPages()`** — создать маппинг `citySlug → text` для всех 28 городов, использовать при генерации.

4. **Обновить `buildPrerenderedContent()`** (строки 305–318) — после заголовков и перед `<nav>` вставлять `<p>${page.text}</p>`, если поле задано.

### Пример результата в HTML

```text
<div id="root">
  <h1>Таможенное оформление импорта</h1>
  <h2>...</h2>
  <h3>...</h3>
  <p>ИННОВЭД выполняет таможенное оформление импорта по всей России...</p>
  <nav><a href="/">Главная</a>...</nav>
</div>
```

### Объём
~120 строк добавлений в одном файле. React заменит содержимое `#root` при монтировании — пользователь ничего не увидит.
